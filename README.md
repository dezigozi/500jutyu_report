# 架装品 500受注レポート

500受注実績Excelファイルを読み込み、部店・顧客・担当者のドリルダウンで売上・粗利を確認できるデスクトップアプリ。

---

## 起動方法

### 通常起動（ダブルクリック）
`受注実績/500受注レポート.vbs` をダブルクリック

### 開発モード（ホットリロードあり）
`lease-report-app/dev.bat` をダブルクリック

---

## できること

### データ読み込み
- ネットワークフォルダ（`\\192.1.1.103\share\特販共有\見積\500データ`）の `500_*.xlsx` を自動検索・読み込み
- 複数年度のファイルを一括読み込み、年度ごとに集計
- 初回読み込み後はキャッシュを作成し、2回目以降は高速起動

### フィルター
- リース会社（略称）でフィルタリング
- 月範囲指定（年度またぎにも対応）

### ドリルダウン表示（2種類）
| モード | 第1階層 | 第2階層 | 第3階層 |
|------|---------|---------|---------|
| 注文者起点 | 部店 | 注文者（得意先担当者） | 顧客名 |
| 顧客起点 | 部店 | 顧客名 + 担当者名表示 | 担当者別内訳 |

### 数値表示
- 売上・粗利の複数年度比較（前年比 % 付き）
- 粗利率の表示
- 金額は万円・億円単位で自動フォーマット

### ピボットテーブル
- リース会社 > 部店 > 注文者 > 顧客 の全組み合わせ一覧
- CSVエクスポート対応

---

## 粗利の計算式

Excelファイルに粗利列はないため、以下の計算式で算出：

```
粗利 = (単価 - 原価) × 受注数量
```

## 列マッピング

| アプリ内項目 | Excelの列 |
|------------|----------|
| リース会社 | AD列（略称） ※列番号固定 |
| 部店 | 部店 |
| 注文者名 | 得意先担当者 |
| 顧客名 | 顧客名_漢 |
| 売上（単価） | 単価 |
| 原価 | 原価 |
| 数量 | 受注数量 |
| 日付 | 登録日 |

## ビルド・開発

```bash
# 開発モード（ホットリロード）
npm run dev

# Reactのビルド（本番用）
npm run build:renderer
```

> ※ コマンドは `lease-report-app` フォルダ内で実行すること

---

## 注意事項

- `500受注レポート.vbs` は ASCII文字のみで記述すること（日本語コメントはエンコードエラーの原因になる）
- ビルド後に `500受注レポート.vbs` から起動する場合は `npm run build:renderer` を実行してからでないと最新の変更が反映されない
