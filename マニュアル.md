# 架装品 500受注レポート — 利用マニュアル

500受注実績の Excel（`500_*.xlsx`）を読み込み、部店・顧客・担当者・品番までドリルダウンして売上・粗利を確認するデスクトップアプリ（Electron）の説明です。

---

## 起動方法

### 本番用（ビルド済みの画面を表示）

- 配布用の **`500受注レポート.vbs`**（親フォルダ側にある場合）をダブルクリックする。  
- または **`npm run build:renderer`** 実行後に **`start.bat`**（`npx electron .`）で起動。

※ ビルド後に VBS から起動する場合は、先に `npm run build:renderer` を実行しないと UI の変更が反映されません。

### 開発モード（ホットリロード・localhost:9000）

次のいずれかで起動します。

| 方法 | 内容 |
|------|------|
| **`dev.bat`** | ダブルクリック → **PowerShell ウィンドウ**（青い画面）が開いて `npm run dev` が走る |
| **`開発で起動.lnk`** | ショートカットをダブルクリック（下記「初回だけ」参照） |
| コマンド | プロジェクトフォルダで `npm run dev` |

`dev.bat` は内部で **`dev.ps1`** を呼び出します。PowerShell は日本語・記号入りのパスを正確に扱えるため、以前の `cmd.exe` による「コマンドとして認識されていません」エラーは出なくなります。起動後はウィンドウをそのまま開いておいてください（開発サーバーが動いています）。

`npm` が見つからない場合は赤字でメッセージが表示されます。その場合は Node.js（https://nodejs.org）をインストール後に再度ダブルクリックしてください。

**ショートカット（アイコン付き）を初回だけ作る場合**

1. **`ショートカット作成.bat`** をダブルクリックする。  
   （または `npm run shortcut:dev`）
2. 同じフォルダに **`開発で起動.lnk`** ができるので、以降はこれをダブルクリックする。

> 画像ファイル（`app-icon.png` など）そのものはアプリではないため、ダブルクリックでは起動しません。ショートカットまたは `dev.bat` を使います。

## データの置き場所と読み込み

- 設定画面で指定したフォルダ内の **`500_*.xlsx`**（`~$` で始まる一時ファイルは除外）を読み込みます。
- 既定の例: `\\192.1.1.103\share\特販共有\見積\500データ`
- 初回はフル解析、以降は **キャッシュ**（ユーザー別データ領域）で高速起動。Excel の更新があれば指紋で再読み込みされます。
- 暗号化ファイルはパスワード解除処理にフォールバックします（`main.js` 内の設定に依存）。

## 画面の機能

### フィルター・表示

- **リース会社**（略称）で絞り込み
- **月範囲**（年度またぎの指定に対応）
- **金額単位**: 円 / 千円
- **粗利表示**の ON/OFF

### ドリルダウン（ダッシュボード）

月別の売上・粗利を、次の階層でたどれます。

- **注文者起点**: 部店 → 注文者 → 顧客 → **品番**
- **顧客起点**: 部店 → 顧客 → 担当者 → **品番**

最終階層（品番）では、品番の**下に商品名**（Excel V列）を表示します。表示は**全角換算で幅15**を超える部分は省略し、マウスを載せると全文を確認できます（同じ品番で複数行ある場合は、先に現れた非空の商品名を採用）。

### 一括網羅レポート（ピボット）

- リース会社 > 部店 > 注文者 > 顧客 の組み合わせ一覧
- **CSV エクスポート**（ピボット用データ）

### PDF

- 画面の印刷／PDF 保存（Electron の `printToPDF`）

## Excel 列とアプリの対応

1行目をヘッダーとして列名を検出し、次の **列番号は診断用に固定オーバーライド**されます。

| アプリでの用途 | 主なヘッダー例 | 固定列（参考） |
|----------------|----------------|----------------|
| リース会社（略称） | 略称 など | AD列（30） |
| 粗利（金額） | 粗利 | AF列（32） |
| 品番 | 品番 | U列（21） |
| 商品名 | 商品名 | V列（22） |
| 受注数量 | 受注数量・受注数・数量 | W列（23） |

その他（部店・顧客名・単価・伝票番号・日付など）はヘッダー文字列のパターンで列を決定します。詳細は `main.js` の `HEADER_PATTERNS` / `OVERRIDE_COLUMNS` を参照してください。

**粗利**は Excel の粗利列（AF列）の数値をそのまま集計に使います。

## 開発・ビルド用コマンド

プロジェクトフォルダ（`lease-report-app`）で実行します。

```bash
# 開発（webpack dev server + Electron --dev）
npm run dev

# 本番用フロントのビルド（build/index.html / bundle.js）
npm run build:renderer

# 開発起動用ショートカットを生成
npm run shortcut:dev

# パッケージ（electron-builder）
npm run build
```

## トラブルシューティング

| 現象 | 確認すること |
|------|----------------|
| データが出ない | 設定のフォルダパス、ネットワーク接続、`500_*.xlsx` の有無 |
| `dev.bat` で「認識されていません」エラー | `dev.ps1` が無い可能性。`lease-report-app` フォルダに `dev.ps1` があるか確認 |
| `dev.bat` で npm が見つからない（赤字） | Node.js 未インストール。https://nodejs.org からインストール後に再実行 |
| 開発が起動しない | Node.js / npm のインストール、`npm install` 済みか |
| 変更が本番起動に出ない | `npm run build:renderer` の実行 |
| キャッシュが古い | アプリ内の「最新データに更新」またはキャッシュ削除（`_clearcache.js` 等、利用中の手順に従う） |

## 補足

- 外部の **`500受注レポート.vbs`** を使う場合、VBS 内は **ASCII のみ**にすると文字化け・実行エラーを避けやすいです。
- **`開発で起動.lnk`** は PC ごとにパスが入るため、リポジトリには含めず `.gitignore` 対象です。別 PC では `ショートカット作成.bat` を再度実行してください。
